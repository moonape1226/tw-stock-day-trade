# 第一性原理分析：tw-stock-day-trade 選股策略

## 1. 從基礎事實出發

當沖要賺錢，必須滿足幾個不可妥協的物理條件：

```
淨利 = (出場價 − 進場價) × 張數 − 成本
     = 可捕捉幅度 × 捕捉率 − 0.45% 來回成本 − 滑價
```

推導出**選股的 4 個第一性必要條件**：
1. **流動性** — 進出不造成滑價吃掉 edge
2. **振幅 > 成本** — 當日波動必須夠付成本且還有剩
3. **方向可辨識** — 需要一個能產生持續性的機制（behavioral 或 informational edge）
4. **進出場時點可執行** — 信號與成交之間的落差可控

下面把 repo 每個濾網對應回這 4 條，看是否站得住腳。

## 2. 各模組 vs 第一性條件

### 2.1 `screen.py` / `screen_v2.py`（候選池篩選）

| 濾網 | 對應條件 | 第一性評估 |
|------|---------|-----------|
| `LIQ_GATE=2e8` 成交額 gate | 流動性 | ✅ 方向對。但用**二元 gate**而非連續函數：2 億跟 5 億的滑價差異被抹平。第一性上滑價是 size 的連續函數。 |
| `MIN_RANGE_PCT=3` / `ATR_LO=2%` | 振幅 > 成本 | ✅ 數學合理：3% × 0.5 捕捉率 = 1.5%，扣 0.45% 成本還有 ~1%。但**捕捉率從未被量測**（見 §3）。 |
| `RVOL≥1.5` 進 A 桶 | 方向可辨識 | ⚠️ 「量能放大 = 注意度 = 動能延續」是行為假設，文獻有支持。但 `rvol = v / avg20_vol` 用**今日累計量**比**全日均量**，未做時段標準化——10 點的 RVOL 看起來永遠偏低，12 點永遠偏高。 |
| `pos≥0.6` 收盤位 | 方向可辨識 | ⚠️ **時間框錯配**：收盤強度是隔日動能訊號，對當沖（當日平倉）意義薄弱。 |
| `ATR_HI=7%` 上限 | 風險控制 | ✅ ATR 過大 → SL 變寬 → 倉位縮水，合理。 |
| `GAP_EXT=4%` 跳空懲罰 | 進場可執行 | ✅ 跳空等於進場已被剝奪一段，合理。 |

v1 → v2 的改進本身符合第一性：v1 的 `log10(turnover)` 在流動性門檻內幾乎是常數（log 在大數區變化小），所以「流動性優先排序」名實不符；v2 改成 GATE 是正確的。

### 2.2 `watch.py`（盤中進場觸發）

| 規則 | 對應條件 | 第一性評估 |
|------|---------|-----------|
| `p > vwap and vd > 0` (f1) | 方向 | ✅ VWAP 是公允值錨，價在上方且上升 = 持有人賺錢、傾向持有。古典且合理。 |
| ORB: `p > or_h and vr > 1.5` (f2a) | 進場時點 | ✅ 開盤區間代表均衡，帶量突破 = 新資訊。結構扎實（Crabel 型）。但 `brk_confirm_bars=2` × 30s = **60 秒延遲**，在快市場中確認完已滑價，README 自己承認 BRK paper P&L 被高估。 |
| PUL: `dp < 0.5% and vr < 0.7` (f2b) | 進場時點 | ⚠️ **這是狀態不是觸發**：條件在「價格正穿越 VWAP 往下、量縮」時就成立，可能在**下跌中途**就點火，而非「拉回止穩後反彈」。第一性上你要的是「拉回完成」的證據（止穩 K、量縮後放量轉折），不是「正在拉回」。 |
| `dVWAP > 2%` → PASS | 進場可執行 | ✅ 不追價，風控合理。 |
| Index filter: `idx.p > prevC and vd > 0` | 方向（環境） | ⚠️ FinMind 指數 snapshot 沒有 `average_price`，所以用**昨收**代替指數 VWAP。README 承認這是較弱代理（單一基準點，易被跳空誤導）。第一性上你想要的是「大盤當日多方結構」，用昨收 + 當日動能只是近似。 |
| R:R 固定 1.5 | 風險控制 | ⚠️ 第一性上 R:R 應與 ATR 適配：高波動股給 1.5R 等於把目標放在常態分佈內，低波動股給 1.5R 可能太遠。固定值不是最適。 |
| 13:15 cutoff | 風險控制 | ✅ 避免留倉 + 末盤流動性差，合理。 |

### 2.3 `chips.py`（三大法人）

第一性上法人買超 = 資訊優勢者的方向。但 repo 把它當**獨立工具**，README 沒提整合路徑，`watch.py` 不讀法人資料。**edge 來源未被接上主流程。**

## 3. 策略的隱含假設（用第一性原理檢驗）

把所有濾網疊起來，repo 隱含的 edge 假設是：

> 「流動性與波動足夠的股票，在價 > VWAP 且 VWAP 上升、大盤同步走強時，若帶量突破開盤區間或量縮拉回 VWAP，會延續方向足以實現 1.5R。」

這是 **trend-following + breakout + mean-reversion-in-trend + regime filter** 的組合，每個元件古典且單獨成立。但第一性檢驗下有 **3 個致命缺口**：

### 缺口 A：Edge 不可量測
`paper_trades.csv` 只記**進場**，不記出場。沒有任何 code 計算實際 P&L 或勝率。
→ **策略是否有 edge 從未被驗證。** 這是第一性最嚴重的問題——沒有回測閉環，所有參數（1.5R、2% EXT、1.5 RVOL）都是猜的。

### 缺口 B：半個機會集被棄用
只做多。README 自己承認「未涵蓋先賣後買空方」。第一性上，當大盤 NG 時整個工具就停擺，但 TW 當沖空方同樣有 edge。

### 缺口 C：screen 與 watch 不相連
- `screen_v2` 用 TWSE openapi（EOD + MIS）
- `watch.py` 用 FinMind（即時）
- 兩者詞彙不同（A/B/C/D 桶 vs BRK/PUL/EXT），沒有程式化銜接
- A 桶條件（RVOL≥1.5, ATR 2-7%）≠ CALL 觸發條件（VWAP 方向 + ORB/PUL）
→ 一檔 A 桶股可能整天不觸發 CALL；一檔 C 桶股可能觸發 CALL。選股與進場**邏輯斷裂**。

## 4. 從第一性原理出的具體建議

依「邊際效益 / 實作成本」排序：

1. **建立出場追蹤 + 回測閉環**（最高優先）
   - watch.py 進場後持續追蹤該 sym 到 13:15，記錄觸 TP / 觸 SL / 收盤價
   - 跑 20 天 paper trades 算淨 R：沒這個，所有參數調整都是盲目

2. **PUL 加止穩確認**（中優先）
   - 現制：`dp<0.5% and vr<0.7` 是狀態
   - 改制：先進入「PUL watch」，再等 `mom>0` 或下根 snapshot 紅翻綠才觸發 CALL
   - 第一性：你要的是「拉回完成」的證據，不是「正在拉回」

3. **RVOL 時段標準化**（中優先）
   - 用 `expected_vol_by_time` 而非 `avg20_vol`：例如 10:00 的預期量 = avg20_vol × 0.25
   - 否則早盤 RVOL 系統性偏低、午盤系統性偏高

4. **R:R 適配 ATR**（低成本）
   - `rr = max(1.0, min(2.5, atr_pct / 1.5))` 之類
   - 高波動股給更寬 R:R，低波動股收緊

5. **把 chips.py 接進 watch.py**（低成本）
   - 盤前跑一次 chips，外資賣超的股票從 watchlist 剔除
   - 第一性：法人方向 = 資訊優勢者方向，是廉價的 edge 來源

6. **screen 與 watch 統一詞彙**（中成本）
   - screen 輸出 A/B/C 桶 + RVOL + ATR，watch 讀取後把 A 桶股優先顯示
   - 否則選股與進場是兩個不相關的策略

7. **加 sector relative strength**（中成本）
   - 一檔漲 3% 在同類漲 5% 時是 laggard，不該選
   - 第一性：當沖要找 leader 不是 laggard

---

## 5. 一句話總結

每個濾網單獨看都古典且方向正確，但**策略沒有閉環驗證 edge**（無出場追蹤）、**選股與進場邏輯斷裂**（screen 與 watch 不相連）、**半個市場被棄用**（只做多）。先補上出場追蹤與回測，其他調整才有意義——否則你不知道現在的 1.5R / 2% EXT / 1.5 RVOL 是不是在調對的方向。
