# 台股當沖選股策略：第一性原理分析

## 結論

這個 repo 的策略本質是「台股偏多當沖動能策略」。

它不是價值選股，也不是長線均值回歸。核心想法是：

> 先找出流動性足夠、波動足夠、今日有異常活躍跡象的股票，再只在大盤順風、個股站上 VWAP 且結構向上時，等待突破或拉回進場。

目前比較像一套「盤中掃描器 + 進場提示器」，還不是完整可驗證的交易系統。原因是它有選股、進場、停損停利建議與 paper log，但缺少完整出場追蹤、滑價、成本、成交可行性與歷史回測。

## 第一性原理

一個當沖策略要成立，至少要同時解決五件事：

1. 流動性：進得去，也出得來。
2. 波動性：價格移動要足以覆蓋交易成本。
3. 方向優勢：不是隨機追價，而是有資金流或趨勢支撐。
4. 進場位置：不能在過度延伸的位置追高。
5. 風險可控：停損距離、報酬風險比與交易成本要合理。

這個 repo 的策略設計大致就是圍繞這五件事。

## screen.py：v1 選股邏輯

`screen.py` 的流程是：

1. 用 TWSE `STOCK_DAY_ALL` 抓上市全市場日線。
2. 用前一交易日成交金額與振幅建立選池。
3. 用 MIS 即時或收盤報價計算今日漲跌幅、日內振幅、收盤位置。
4. 偏好多頭強勢股：收紅、收在高檔、有量、有振幅。

核心條件：

- 前日成交金額 >= 2 億。
- 前日振幅 >= 3%。
- 今日漲跌幅 >= 0%。
- 今日收盤位置 >= 當日區間 60%。
- 今日振幅 >= 3%。

排序分數：

```text
score = 收盤位置 * 50 + min(今日漲幅, 9.9) * 4 + log10(成交金額)
```

第一性原理評估：

- 合理處：它知道短線交易需要流動性、波動與收盤強度。
- 問題處：`log10(turnover)` 幾乎是常數，無法真正代表「流動性優先」。
- 問題處：`min(pct, 9.9) * 4` 會獎勵接近漲停的股票，容易把策略推向追高。
- 問題處：成交金額和振幅同時做 gate 又進排序，訊號有重複計分的味道。

因此 v1 比較像「強勢股排行榜」，不夠像「可執行的當沖候選池」。

## screen_v2.py：較合理的候選池

`screen_v2.py` 修正了 v1 的主要偏誤。

它把流動性從排序因子改成 gate：

```text
avg20 成交額 >= 2 億
```

這符合交易第一性原理：流動性是能不能交易的必要條件，不是越大就越有 alpha。

v2 的排序主要看：

- RVOL：今日成交量 / 20 日均量。
- ATR%：ATR20 / 昨收，判斷波動是否適合當沖。
- 收盤強度：收在日內區間高檔，但降權。
- extension penalty：跳空太大、漲停、收太高且漲太多會扣分。

分桶邏輯：

- A：核心可執行，RVOL 夠高、未過度延伸、ATR 在可接受區間。
- B：動能延伸，可能強但要等拉回。
- C：邊際。
- D：排除，漲停、流動性不足或 RVOL 不足。

第一性原理評估：

- 這比 v1 健康，因為它不再把高成交金額誤當 alpha。
- RVOL 比單純成交金額更接近「今日資金異常活躍」。
- ATR% 比單日振幅更適合衡量平常是否有足夠可交易波動。
- extension penalty 能避免把最危險的追高標的排到最前面。

## watch.py：盤中進場策略

`watch.py` 是實際盤中監控面板。

它使用 FinMind 即時 snapshot，對每檔股票計算：

- 現價。
- VWAP。
- VWAP 方向。
- 現價距 VWAP 百分比。
- 開盤區間高低。
- 量比。
- 與前次更新相比的短線動能。
- 資料年齡。

最核心的個股方向條件：

```text
price > VWAP 且 VWAP direction > 0
```

也就是只做 VWAP 上方、VWAP 向上的股票。

大盤條件：

```text
加權指數 > 昨收
且短線方向向上
且沒有快速下跌 crash
```

這代表策略只在市場順風時做多。

## 進場模式

### BRK：突破

BRK 條件：

```text
price > opening range high
且 volume_ratio > breakout_volume_ratio
且連續確認次數 >= brk_confirm_bars
```

預設參數：

```json
{
  "breakout_volume_ratio": 1.5,
  "brk_confirm_bars": 2
}
```

停損停利：

- Entry：開盤區間高點。
- SL：開盤區間低點。
- TP：Entry + 1.5R。

第一性原理：

BRK 在賭「早盤形成的供需區間被放量突破後，價格會延續」。這是典型 opening range breakout。

主要風險：

- 如果開盤區間太大，停損距離會過寬。
- 如果突破發生在大幅延伸後，容易買在日內高點。
- 若沒有逐筆或 kbar 驗證，連續兩次 snapshot 確認不等於真正穩定突破。

### PUL：拉回

PUL 條件：

```text
abs(price - VWAP) / VWAP < pullback_vwap_range_pct
且 volume_ratio < pullback_volume_ratio
```

預設參數：

```json
{
  "pullback_vwap_range_pct": 0.5,
  "pullback_volume_ratio": 0.7
}
```

停損停利：

- Entry：VWAP 附近。
- SL：VWAP 下方約 1%，或至少 5 ticks。
- TP：Entry + 1.5R。

第一性原理：

PUL 在賭「多頭日內結構中，價格回到 VWAP 附近且量縮，代表賣壓暫時減弱，後續可能恢復上攻」。

這比 BRK 更重視進場位置，通常停損距離也比較可控。

## 過度延伸過濾

`watch.py` 有一個簡單但重要的過濾：

```text
dVWAP > 2% => PASS
```

意思是現價距 VWAP 超過 2% 就不追。

第一性原理：

VWAP 是當日市場平均持倉成本的近似。價格離 VWAP 太遠，代表短線買方已經付出很多溢價。此時再追，勝率不一定低，但盈虧比通常變差，因為合理停損會變寬，回歸 VWAP 的風險變高。

## chips.py：法人籌碼輔助

`chips.py` 不是即時進場策略，而是監控清單的輔助過濾。

它抓 TWSE T86 三大法人買賣超，對 `config.json` 裡的股票標記：

- 外資買超。
- 投信買超。
- 外資與投信皆未買超。

規則是：

```text
外資或投信任一買超即保留
```

第一性原理：

法人買超可以視為中期資金偏好，但對當沖進場的即時性有限。它比較適合拿來調整觀察清單，不適合直接當盤中買進訊號。

## 目前策略真正賭的是什麼

整體策略的 alpha 假設可以寫成：

> 在台股上市股票中，若一檔股票具備足夠流動性、平常波動適中、今日 RVOL 明顯放大，且盤中價格站上 VWAP、VWAP 向上、加權指數順風，則它在突破開盤區間或回測 VWAP 後，有較高機率出現日內延續行情。

這個假設並非不合理，但目前 repo 還沒有證明它有正期望值。

## 主要風險

### 1. 開盤區間可能失真

`watch.py` 的 opening range 是靠程式啟動後累積 9:00 到 9:30 的 high/low。

如果程式不是 9:30 前啟動，會用當下 snapshot 的 high/low 初始化並凍結。這可能讓 BRK 變成「突破當日高」或其他非原始 opening range 的定義。

這會影響 BRK 訊號的一致性。

### 2. 只有進場，沒有完整出場系統

README 明確寫到進場後管理完全人工。

目前面板會給：

- Entry。
- SL。
- TP。
- R:R。

但它不追蹤持倉、不提醒出場、不計算真實損益。這使得 paper log 只能記錄進場訊號，不能直接評估策略期望值。

### 3. 成本沒有進入決策

`screen_v2.py` 顯示當沖來回成本約 0.45%，但這只是提示，沒有進入 score 或交易可行性判斷。

對當沖來說，0.45% 是很大的 hurdle。若平均停利空間只有 1% 左右，成本與滑價會顯著侵蝕期望值。

### 4. 多頭單邊策略

策略只做多，且大盤弱時不做。

這是保守設計，但也表示：

- 空頭日沒有空方策略。
- 盤整日可能頻繁 WAIT 或 PASS。
- 強勢股在尾端階段可能出現假突破。

### 5. 沒有產業與題材群聚

短線台股常有族群輪動。

目前策略只看單股技術與大盤，沒有判斷：

- 同族群是否同步放量。
- 領漲股與補漲股關係。
- 題材是否集中。
- 是否只有單一個股異常。

這可能少了一層有效的盤中確認。

### 6. 上櫃、禁現沖、處置限制尚未完整處理

`screen_v2.py` 註解中也提到尚未涵蓋：

- 上櫃 TPEx。
- 官方禁當沖清單。
- kbar 回測。
- 空方策略。

這些都會影響實際可交易性。

## 建議的驗證方向

最有價值的下一步不是再加更多條件，而是先做回測與交易紀錄閉環。

建議優先順序：

1. 建立 1 分 K 或 tick replay 回測。
2. 固定 BRK/PUL 定義，避免啟動時間影響 opening range。
3. 將 Entry、SL、TP、13:15 出場規則程式化。
4. 加入手續費、證交稅、滑價。
5. 分別統計 BRK 與 PUL 的勝率、平均 R、最大回撤。
6. 分桶比較 A/B/C 候選池後續盤中表現。
7. 驗證不同大盤狀態下的表現差異。

## 最終評價

目前策略方向合理，尤其 `screen_v2.py` 比 `screen.py` 更接近交易第一性原理。

它的優點：

- 有流動性 gate。
- 有 RVOL 判斷。
- 有 ATR 適配。
- 有 VWAP 結構。
- 有大盤濾網。
- 有過度延伸懲罰。
- 有停損停利框架。

它的不足：

- 沒有完整回測。
- 沒有真實出場追蹤。
- 成本與滑價未納入期望值。
- opening range 依賴啟動時間。
- 題材與產業同步性未納入。
- 目前只能證明「規則看起來合理」，還不能證明「策略有正期望值」。

一句話總結：

> 這是一個架構合理的台股偏多當沖掃描器，但要升級成交易系統，下一步必須補上 kbar 回測、完整出場、成本滑價與真實績效統計。
